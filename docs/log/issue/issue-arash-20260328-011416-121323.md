## Bug: projio note_read tool requires `path` param but note_list only returns metadata

### Reproduction
1. Call `mcp__projio__note_list(note_type="idea")` — returns notes with `path`, `timestamp`, `title`, etc.
2. Call `mcp__projio__note_read(note_type="idea", note_id="20260328-010019-846142")` — fails with:
   ```
   3 validation errors for call[note_read_tool]
   path: Missing required argument
   note_type: Unexpected keyword argument
   note_id: Unexpected keyword argument
   ```

### Expected
`note_read` should accept `note_type` + `note_id` (or just `note_id`) like the other note tools, and resolve to the file internally. Alternatively, accept the `path` returned by `note_list`.

### Workaround
Read the file directly via the Read tool using the `path` from `note_list`.
