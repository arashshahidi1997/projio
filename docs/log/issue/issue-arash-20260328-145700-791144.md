## Migrate graph.py from labpy sutil/biblio to projio biblio

`sutil/biblio/graph.py` in labpy contains `expand_openalex_reference_graph()` — an iterative multi-hop reference graph expansion via OpenAlex, seeded from known citekeys. This is the only unique functionality in labpy's legacy biblio module that projio doesn't yet cover.

### Task
1. Read `/storage2/arash/sirocampus/code/labpy/src/sutil/biblio/graph.py` to understand the API
2. Read the projio biblio package structure to find the right location (likely under the biblio package)
3. Port the graph expansion logic into projio's biblio package, adapting it to use projio's existing OpenAlex client and citekey resolution
4. Add an MCP tool `biblio_graph_expand(citekeys, max_hops)` or similar that exposes this functionality
5. Write a basic smoke test

### Context
- labpy audit confirmed this is the only piece worth migrating before deleting legacy modules
- `paper_absent_refs()` in projio handles single-paper unresolved refs but not multi-hop graph traversal
- After this migration, labpy's entire sutil/biblio/ can be deleted
