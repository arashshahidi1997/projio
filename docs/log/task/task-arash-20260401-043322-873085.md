# Flatten pipe/flow to flow-only: registry schema and core model

## Goal

Change pipeio's addressing model from `(pipe, flow)` to flow-only. The `pipe` concept is replaced by an optional `derivative` field on FlowEntry that defaults to the flow name.

## Context

Currently all pipeio tools require `(pipe, flow)` to address a flow. In practice, 7 out of 8 pipes have exactly one flow ÔÇö the pipe level is redundant. This task changes the registry schema and core model. Subsequent tasks will update MCP tools, CLI, docs, and wrappers.

**Design decisions (already validated):**
- Registry key changes from `"pipe/flow"` to just `"flow"` (flow names must be globally unique)
- `FlowEntry.pipe` Ôćĺ `FlowEntry.derivative` (optional, defaults to flow name)
- `derivative` determines the output directory: `derivatives/{derivative}/`
- `FlowEntry.code_path`, `config_path`, `doc_path` remain as-is (filesystem doesn't change)
- No physical file moves ÔÇö only the addressing model changes

## Prompt

> Refactor the pipeio registry schema to use flow-only addressing.
>
> ### Files to modify
>
> **`packages/pipeio/src/pipeio/registry.py`:**

> 2. `PipelineRegistry.flows` dict: change key from `"pipe/flow"` to just the flow name
> 3. `list_pipes()` Ôćĺ rename to `list_derivatives()`, returns unique effective derivative names

> 5. `get(pipe, flow)` Ôćĺ change to `get(flow)`, remove auto-selection logic
> 6. `remove(pipe, flow)` Ôćĺ change to `remove(flow)`
> 7. `scan()` ÔÇö update `_discover_flows`: key by flow name, set `derivative` from the parent pipe directory name
> 8. `to_yaml()` / `from_yaml()` ÔÇö update serialization. Keep backward compat: if old YAML has `"pipe/flow"` keys, parse them (set `derivative` from pipe part, key by flow name)
> 9. `validate()` ÔÇö check flow name uniqueness instead of pipe/flow
