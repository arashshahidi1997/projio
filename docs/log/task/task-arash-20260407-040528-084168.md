Done. Here's a summary of the fix:

**Problem:** When `env` is set, `python_bin` was constructed as a single string `"conda run -n cogpy python"`, which `subprocess.run()` treated as a single executable path (element 0 of the list), causing `FileNotFoundError`.

**Fix across 3 files:**

1. **`src/projio/mcp/pipeio.py:1120`** — Changed `python_bin` from f-string to list: `[conda, "run", "-n", env, "python"]`

2. **`packages/pipeio/src/pipeio/notebook/lifecycle.py`** — Added `_python_prefix()` normalizer that handles `str | list[str] | None`. Updated `_require_jupytext()` and `_jupytext()` to use `[*prefix, "-m", "jupytext", ...]` instead of `[python_bin, "-m", ...]`. Updated type annotations on all functions that accept `python_bin`.

3. **`packages/pipeio/src/pipeio/mcp.py`** — Added same `_python_prefix()` helper. Updated `_has_papermill()` and the papermill cmd construction in `mcp_nb_exec` to use `[*prefix, ...]` pattern. Updated `python_bin` type annotation on `mcp_nb_exec`.

Backward compatible — plain string `python_bin` (e.g. `"/usr/bin/python3"`) still works via the normalizer.
