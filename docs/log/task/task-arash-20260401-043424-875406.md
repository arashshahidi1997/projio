# Flatten pipe/flow: update pipeio CLI commands

## Goal

Update all pipeio CLI commands to use flow-only addressing. Remove `pipe` positional arguments and `--pipe` flags.

## Context

Prior tasks changed the registry schema (FlowEntry.pipe → derivative) and MCP functions (dropped pipe params). This task updates the CLI layer in `packages/pipeio/src/pipeio/cli.py`.

## Prompt

> Update all CLI commands in `packages/pipeio/src/pipeio/cli.py` to use flow-only addressing.
>
> ### Commands to update
>
> **Flow commands (argparser + handler):**
> - `pipeio flow list [pipe]` → `pipeio flow list [--derivative D]`
> - `pipeio flow new <pipe> <flow>` → `pipeio flow new <flow> [--derivative D]`
> - `pipeio flow fork <pipe> <flow> <new_flow>` → `pipeio flow fork <flow> <new_flow> [--derivative D]`
> - `pipeio flow status <flow>` → already takes just flow via pipe auto-detection, simplify
> - `pipeio flow path <flow>` → same
> - `pipeio flow config <flow>` → same
> - `pipeio flow deriv <flow>` → same
> - `pipeio flow run <flow>` → same
> - `pipeio flow log <flow>` → same
> - `pipeio flow mods <flow>` → same
>
> **Registry commands:**
> - `pipeio registry deregister <pipe> <flow>` → `pipeio registry deregister <flow>`
>
> **Notebook commands:**
> - `pipeio nb lab --pipe P --flow F` → `pipeio nb lab [--flow F]` (drop --pipe)
> - `_detect_flow_from_cwd()` → update to use new registry.get() signature
>
> ### Handler functions to update
> Each `_cmd_*` function that calls `registry.get(pipe, flow)` or `mcp_*(root, pipe=, flow=)` needs updating.
>
> The `_cmd_flow_new` scaffold should set `derivative` in the notebook.yml and Snakefile comments instead of `pipe/flow`.
>
> ### Also update:
> - `_detect_flow_from_cwd()` — match on `entry.name` (already does) but use new registry API
> - `_load_registry()` helper if it references pipe
> - `_cmd_registry_scan` — update summary output
>
> ### Constraints
> - Only modify `packages/pipeio/src/pipeio/cli.py`
> - Run `PYTHONPATH=src python -m pytest tests/ -q` to verify
> - Save with `datalad_save`

## Acceptance Criteria

- [ ] No `pipe` positional arg in any CLI command
- [ ] `--derivative` optional flag where needed
- [ ] `pipeio flow new <flow>` works
- [ ] `pipeio registry deregister <flow>` works
- [ ] `pipeio nb lab` auto-detection still works
- [ ] All tests pass
