# Pipeline Context API (Python)

This document specifies the **notebook-facing pipeline bootstrap API** implemented in `code/utils/io/`.

Goal: let notebooks and one-off scripts do:

- load a pipeline flow config (`config.yml`) via a canonical registry
- generate Snakebids inputs once
- construct product paths via existing registries (`sutil.bids.paths.BidsPaths`)
- handle “session selection”, “bundles”, and entity parsing consistently

Non-goals:

- replacing Snakemake or per-pipeline `config.yml` contracts
- re-implementing PyBIDS layouts; this API uses Snakebids + filename entity parsing

---

## Canonical registry files

### Pipe/flow/mod registry

- Canonical: `workflow/pipe_flow_mod_registry.yml`
- Generated by: `code/utils/cli/gen_pipe_flow_mod_registry.py`

This registry is **discovery + path mapping** (docs + code), and includes a per-flow config pointer:

- `pipes.<pipe>.flows.<flow>.code.config_path` (repo-relative, typically `code/pipelines/<pipe>/<flow>/config.yml`)

## Public API surface

All public notebook-facing imports are exposed from:

- `utils.io` (`code/utils/io/__init__.py`)

### Registry/config helpers (`utils.io.registry`)

#### `load_pipeline_registry(path: Path | None = None) -> dict`

- Loads the pipe/flow registry payload.
- Default path: `workflow/pipe_flow_mod_registry.yml`.

#### `PipelineRegistry.load_default() -> PipelineRegistry`

- Convenience wrapper around `load_pipeline_registry()`.

#### `PipelineRegistry.get(pipe: str, flow: str | None = None) -> PipelineEntry`

Returns a `PipelineEntry` with:

- `pipe`, `flow`
- `meta`: raw flow metadata
- `config_path`: resolved `config.yml` path

Flow resolution rules when `flow=None`:

- if the pipe has exactly one flow, select it
- else if a flow exists with the same name as the pipe, select it
- else raise (require an explicit `flow=...`)

#### `PipelineEntry.load_config() -> dict`

- Loads the flow’s `config.yml`.
- Adds `_cfg_path` (repo-relative) for debugging.

#### `get_pipeline_config(pipe: str, flow: str | None = None, registry: dict | None = None) -> dict`

- Loads and returns `config.yml` (a mapping).
- Adds `_cfg_path` (repo-relative).

## Pipeline bootstrap (`utils.io.context`)

### `PipelineContext`

A runtime context for a single pipeline flow.

#### Constructors

##### `PipelineContext.from_registry(pipe: str, flow: str | None = None, repo_root: Path | None = None, registry: PipelineRegistry | None = None) -> PipelineContext`

1. Resolves the flow `config.yml` via `PipelineRegistry`
2. Loads the YAML config
3. Delegates to `from_config(...)`

##### `PipelineContext.from_config(pipe: str, flow: str | None, cfg: Mapping[str, Any], cfg_path: str | Path, repo_root: Path | None = None) -> PipelineContext`

Bootstraps the runtime objects:

- Snakebids inputs via `snakebids.generate_inputs(...)`
  - uses `cfg["bids_dir"]` if present, else `cfg["input_dir"]`
  - uses `cfg["pybids_inputs"]`
- Path resolvers using `sutil.bids.paths.BidsPaths`:
  - `in_paths` if `cfg["input_registry"]` + (`cfg["input_dir"]` or `cfg["bids_dir"]`) exist
  - `out_paths` if `cfg["registry"]` + `cfg["output_dir"]` exist
  - `extra_in_paths` for any pairs:
    - `input_dir_<name>` and `input_registry_<name>`

Dependencies:

- `snakebids`
- `sutil` (`sutil.bids.paths.BidsPaths`)
- `PyYAML`

#### Attributes

- `pipe`, `flow`
- `cfg_path`: absolute `config.yml` path
- `cfg`: parsed config mapping
- `inputs`: Snakebids inputs
- `in_paths`: `BidsPaths | None`
- `out_paths`: `BidsPaths | None`
- `extra_in_paths`: `dict[str, BidsPaths]`

#### Path API

##### `pattern(group: str, product: str) -> str`

- Returns the pattern string for an output product (with `{entity}` placeholders).
- Requires `out_paths` to be configured.

##### `path(group: str, product: str, **entities) -> Path`

- Returns a concrete `Path` with entities applied.
- Requires `out_paths`.

##### `have(group: str, product: str, **entities) -> bool`

- `True` if the resolved path exists on disk.

#### Introspection

##### `groups() -> list[str]`

- Output registry families (top-level keys of `cfg["registry"]`).

##### `products(group: str) -> list[str]`

- Output members for the family.

#### Sessions

##### `session(**entities) -> Session`

- Wraps entities and delegates to `PipelineContext.path(...)`.

##### `iter_sessions(base_input: str, entities: Sequence[str]) -> list[Session]`

- Enumerates consistent entity rows using Snakebids `inputs[base_input].zip_lists`.
- Each row becomes a `Session`.

#### Expansion helpers

##### `expand(group: str, product: str, filter: Mapping[str, Any] | None = None, max_n: int | None = None) -> list[Path]`

- Expands an output pattern over the appropriate Snakebids base input (from the family’s `base_input`).
- Supports filtering using Snakebids component `.filter(...)`.

##### `parse_entities(path: str | Path) -> dict[str, str]`

- Delegates to `utils.bids.parse_entities(...)` (PyBIDS filename parsing).

#### Bundles

##### `bundle_members(bundle: str, group: str | None = None) -> list[str]`

Resolution order:

1. `cfg["bundles"][bundle]["members"]` (preferred; if present)
2. `cfg["_member_sets"][bundle]` keys (common in existing configs via YAML anchors)
3. else error

## Session (`utils.io.session`)

### `Session`

An immutable container of entities + a `PipelineContext`.

#### `get(group: str, product: str, **overrides) -> Path`

- Returns a concrete product path.
- `overrides` replace stored entity values.

#### `have(group: str, product: str, **overrides) -> bool`

- Existence check.

#### `bundle(group: str, bundle: str, **overrides) -> dict[str, Path]`

- Returns `{member_name: Path}` for all members in the bundle.
- Uses `PipelineContext.bundle_members(...)` + `PipelineContext.path(...)`.

---

## Notebook bootstrap (`utils.io.bootstrap`)

### `notebook_bootstrap(add_code: bool = True, add_lib_src: bool = True, extra: Iterable[str | Path] = ()) -> Path`

Adds common repo paths to `sys.path`:

- `<repo>/code` (so `import utils` works)
- `<repo>/code/lib/*/src` (so in-repo libs are importable)

Returns the resolved repo root.

---

## Example

```python
from utils.io import notebook_bootstrap, PipelineContext

notebook_bootstrap()

ctx = PipelineContext.from_registry("preprocess", flow="ecephys")
sess = ctx.session(subject="test", session="test", task="pre", acq="lshank", recording="lf")

pre_all = sess.get("all", "all")
bundle = sess.bundle("preprocess", "ecephys_bundle")
```
